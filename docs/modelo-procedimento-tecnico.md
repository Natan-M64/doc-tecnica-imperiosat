## 2. Objetivo

Descrever de forma simples o objetivo do procedimento.

Exemplo:

> Este procedimento tem como objetivo orientar a execução de uma rotina técnica interna, garantindo que a atividade seja realizada de forma padronizada, segura e compreensível para os colaboradores.

---

## 3. Quando utilizar

Informar em quais situações este procedimento deve ser utilizado.

Exemplo:

- Durante uma manutenção técnica.
- Ao configurar um novo serviço.
- Ao realizar atendimento interno.
- Ao documentar uma rotina recorrente.

---

## 4. Requisitos necessários

Listar os recursos necessários para executar o procedimento.

Exemplo:

- Acesso autorizado ao sistema.
- Computador com navegador atualizado.
- Conta de usuário com permissão adequada.
- Documentação complementar, se houver.

---

## 5. Passo a passo

### Etapa 1 - Verificação inicial

Descrever a primeira verificação necessária antes de executar o procedimento.

### Etapa 2 - Execução da atividade

Descrever como a atividade deve ser realizada.

### Etapa 3 - Conferência

Descrever como validar se o procedimento foi executado corretamente.

### Etapa 4 - Registro

Informar onde registrar o procedimento executado, caso seja necessário.

---

## 6. Cuidados importantes

- Não compartilhar senhas ou dados sensíveis em documentos públicos.
- Validar as informações antes de concluir o procedimento.
- Registrar alterações relevantes.
- Solicitar apoio técnico em caso de dúvida.

---

## 7. Resultado esperado

Ao final do procedimento, espera-se que a atividade esteja concluída, registrada e compreensível para futuras consultas.
